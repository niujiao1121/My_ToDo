# AI TODO 创建 - 示例集合

本文档提供丰富的 AI TODO 创建示例，帮助你更好地使用这个功能。

## 📝 基本使用

### 示例 1：简单日常任务

**输入**：
```
明天前完成周报
```

**AI 生成的 TODO**：
- **标题**：完成周报
- **描述**：完成本周工作周报的撰写和提交
- **截止日期**：[明天的日期]
- **优先级**：medium
- **模块**：docs
- **验收标准**：
  - 总结本周主要工作内容
  - 列出下周工作计划
  - 提交到指定系统

---

## 🚀 技术开发任务

### 示例 3：后端 API 开发

**输入**：
```
紧急修复支付接口超时问题，后端优化，3天，下周五完成
```

- **优先级**：critical
- **模块**：backend
- **预计工作量**：24 小时（3天）

- **任务清单**：
  1. 分析接口性能瓶颈
  2. 优化数据库查询
  3. 添加缓存机制
  4. 代码优化和重构
  5. 编写单元测试
  6. 部署和验证

## 💡 最佳实践

### ✅ 好的输入示例

1. **明确具体**：
   ```
   优化用户列表页面加载速度，目标从 3 秒降到 1 秒以内，前端优化，下周五完成
   ```

2. **包含关键信息**：
   ```
   紧急修复订单支付失败的 bug，影响所有用户，后端问题，今天必须完成
   ```

3. **合理的工作量估算**：
   ```
   实现商品搜索功能，包括关键词搜索和分类筛选，预计 3 天
   ```

### ❌ 不够理想的输入

1. **太过简单**：
   ```
   做个功能
   ```
   > AI 无法理解具体要做什么

2. **信息不明确**：
   ```
   有个 bug 需要修
   ```
   > 缺少具体的 bug 描述和位置

3. **表达模糊**：
   ```
   那个东西尽快搞定
   ```
   > 没有明确的任务目标和时间

### 🎯 输入技巧

1. **使用关键词**：
   - 时间：今天、明天、下周、本月底
   - 优先级：紧急、重要、一般、不急
   - 模块：前端、后端、数据库、设计
   - 工作量：X 天、X 小时

2. **结构化描述**：
   ```
   任务名称 + 具体要求 + 模块 + 时间 + 优先级
   ```

3. **避免歧义**：
   - 使用明确的日期而不是"过几天"
   - 使用具体的数值而不是"很多"、"一些"
